How It Works

The algorithm powering your schedule

01

Knapsack Problem

Given a set of subjects (items) each requiring a certain number of hours (weight) and offering a study benefit (value), find the optimal subset that fits within your available time (capacity).

02

Benefit Formula

Each subject gets a benefit score computed as:
(priority × 30) + (difficulty × 15)
High priority + hard subjects are worth studying most.

03

DP Table

A 2D table dp[i][w] stores the max benefit using first i subjects within w half-hours. Each cell: max(skip, take).

04

Backtracking

After filling the table, we trace backwards from dp[n][W] to find exactly which subjects were selected — giving you the optimal set.

05

Complexity

Time: O(n × W) where W = total half-hours.
Space: O(n × W) for the DP table.
Practical and fast even for 20+ subjects.

06

Why 0/1?

Each subject is either fully studied or skipped — you can't study half a subject. This "all or nothing" constraint is exactly the 0/1 variant of the knapsack problem.
